# 🚀 런팟(RunPod) 새 인스턴스 부활 가이드 (V2 최적화 반영)

내일 새로운 런팟 인스턴스를 하나 팠을 때, **오늘 세팅한 완벽한 상태(I2V + GGUF)**로 단숨에 복구하기 위한 체크리스트입니다.

---

## 1️⃣ [STEP 1] 런팟 생성 직후 할 일 (기본 환경 세팅)
새 인스턴스의 **IP 주소**와 **SSH 포트 번호**를 확인합니다.

1. 로컬 프로젝트 폴더의 `런팟_컨피_원클릭_시동.command` 파일을 편집기(VScode 등)로 엽니다.
2. 아래 변수들의 값을 새로 파신 런팟 정보로 교체합니다.
   - `RUNPOD_IP="새로운 IP"`
   - `RUNPOD_SSH_PORT="새로운 포트"`
3. 수정한 후, `런팟_컨피_원클릭_시동.command` 파일을 더블클릭하여 실행합니다.
   > **결과**: 이 스크립트가 커스텀 노드(GGUF 플러그인 등), 모델들(Flux GGUF, Qwen4B, VAE 등), 그리고 ComfyUI 최신화까지 **알아서 전부 다운로드하고 세팅**해 줍니다. 

---

## 2️⃣ [STEP 2] I2V (비디오 생성) 복구 세팅
`런팟_컨피_원클릭_시동`이 끝나고 `[성공]` 메시지가 뜨면 런팟 브라우저(http://localhost:8181)로 접속합니다.

1. 빈 화면에서 우측 메뉴의 **[Load]** 버튼을 누릅니다.
2. 파일 선택 창에서 로컬 컴퓨터의 `core_v2/LTX_TURBO_I2V_API.json` 파일을 찾아 올립니다.
   > **결과**: 이 JSON 파일에는 오늘 우리가 잡아놓은 **"해상도 720px 고정"**, "피아노 박멸 노드" 등 완벽하게 튜닝된 비디오 생성 노드들이 모두 들어있습니다. 서버에 그대로 이식됩니다.
3. 로드된 상태에서 화면 우측 메뉴의 **[Save (API Format)]**을 눌러서 로컬 어딘가에 한 번 저장해 줍니다. (ComfyUI가 이제 해당 세팅을 정상 인식하게 만드는 절차)

---

## 3️⃣ [STEP 3] 즉시 돌려보기
이제 어제(오늘)와 동일하게 일하시면 됩니다. 프롬프트나 모든 명령어 로직은 로컬 파이썬 코드에 남아 있습니다.

- **이미지 생성할 때**: `02-3-20_오디오_기준_이미지_자동생성.command` 그냥 실행
- **비디오 생성할 때**: `02-3-30_선택_이미지_비디오_생성.command` 그냥 실행
   - `02-145...py` 스크립트 안에 오늘 극찬받은 **"영문 베이스 + 한글 대사 립싱크" + 배기(Batch) 시스템**이 그대로 보존되어 있습니다.

---
