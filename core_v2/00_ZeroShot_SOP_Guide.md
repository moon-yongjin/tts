# 🎙️ [SOP] 제로샷 음성 클로닝 가이드 및 표준 실행 지침

사용자님의 최적화된 로컬 맥(Apple Silicon) 연동 환경을 기반으로 한 **제로샷(Zero-Shot) 음성 복제 표준 가공 지침**입니다. 향후 모든 유사 작업은 본 가이드의 아키텍처를 준수합니다.

---

## 📌 [기본 원칙]
1. 제로샷 목소리 추출은 **로컬 8-bit Base 모델**을 기본 탑재합니다.
2. 대본에 있는 **숫자(`40`, `123`)는 반드시 자동 한글화 치환기**를 거쳐 읽기 오류를 원천 차단해야 합니다.
3. 원격 클라우드 다운로드 대신 **로컬 캐시 에셋을 최우선 연동**하여 연산 속도를 보장합니다.
4. **마침표(` . `) 전처리**: 생성 모델 특성상 마침표에서 피치 드롭 오류가 발생할 수 있으므로, 반드시 `normalize_text()`를 통해 마침표를 **쉼표(`,`)로 치환**하여 자연스러운 쉼표 호흡 처리를 거칩니다.

---

## 🏃‍♂️ [표준 실행 자동화 바이블]

### 1단계: 자동 등록 및 에셋 가공 (All-in-One)
*   **파일**: `1-3-106_ZeroShot_CustomRef.py` 사용
*   **자동 루틴**:
    *   **추출**: `ffmpeg`를 통해 **WAV 24000Hz Mono**로 물리적 가공 (통합 저장소 `voices/Reference/` 이동)
    *   **전사**: `Mlx-Whisper`를 돌려 사용자가 입력한 대사가 아닌, **실제 오디오와 1:1 일치하는 대사(.txt)**로 자동 보정
    *   **등록**: 연관 파일 3종에 다음 순번으로 즉시 영구 등록

### 2단계: 최적화 V3 템플릿 준수
*   **핵심 로직**: 마침표 2개(`.. `) 공백 처리 및 숫자 한글화 필수 탑재
*   **모델**: `Qwen3-TTS-...-8bit` 로컬 가중치 사용

### 3단계: 통합 파이프라인 가동
*   **파일**: `1_1통합파이프뉴_듀얼_고품질.command` 실행
*   **확인**: `1-3-104_Dual_Split_Master.py` 로드 시 등록한 신규 번호가 메뉴에 출력되는지 확인
